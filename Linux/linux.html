What is Linux?

Linux is a free and open-source operating system based on Unix. It powers everything from personal computers and smartphones to servers and cloud infrastructure. Linux is known for its stability, performance, and security.

Why Linux is Important in DevOps

Most servers, containers, and CI/CD pipelines in DevOps are built on Linux. Understanding Linux is essential for:

  • Managing cloud-based systems (like AWS EC2)
  • Automating deployments and configurations
  • Writing scripts for system tasks
  • Troubleshooting performance issues

Basic Linux Commands

  • ls – list directory contents
  • cd – change directory
  • cp, mv, rm – copy, move, and delete files
  • chmod, chown – change permissions and ownership
  • top, ps – monitor system processes

File System Structure

The Linux file system is hierarchical. Key directories include:

  • / – root of the file system
  • /home – user directories
  • /etc – configuration files
  • /var – log files
  • /usr – user applications

User & Permission Management

Linux uses a permission model (read, write, execute) for user, group, and others.

  • Create users: useradd
  • Switch users: su or sudo
  • Check permissions: ls -l

Shell Scripting

Bash scripting is used to automate tasks such as deployments, backups, and monitoring. 

#!/bin/bash
echo "Hello from Linux script"

Package Management

Linux uses package managers to install and manage software:

  • Debian/Ubuntu: apt (e.g. sudo apt install nginx)
  • RHEL/CentOS: yum or dnf

Troubleshooting Tips

  • Check system logs: journalctl or /var/log
  • Inspect services: systemctl status nginx
  • Monitor disk: df -h, du -sh *
